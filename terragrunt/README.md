# icecube.dog Terragrunt

This directory contains the shared `kevcube` AWS account infrastructure that is
relevant to this repo, plus the `icecube.dog` site infrastructure moved from
`~/code/react-native-web/openteller/terragrunt/`.

Included units:

- Shared account infrastructure:
  - `aws/kevcube/global/iam/groups/devs`
  - `aws/kevcube/global/iam/users`
  - `aws/kevcube/global/security/github-oidc`
  - `aws/kevcube/eu-central-1/security/kms/sops-key`
- Route53 hosted zone: `aws/kevcube/global/dns/icecube.dog/zone`
- Route53 records: `aws/kevcube/global/dns/icecube.dog/records`
- ACM certificate for CloudFront: `aws/kevcube/us-east-1/security/acm/icecube.dog`
- S3/CloudFront CDNs: `aws/kevcube/eu-central-1/s3-cdn` and `aws/kevcube/eu-central-1/kevin-twa`
- `kb.icecube.dog` origin bucket: `aws/kevcube/us-east-2/s3/kbmarriage.icecube.dog`
- `kb.icecube.dog` CloudFront: `aws/kevcube/us-east-1/cloudfront/kbmarriage.icecube.dog`

`openteller.com`-specific infrastructure stays in the `openteller` repo.

The layout was kept the same so the remote state keys generated by
`path_relative_to_include()` stay stable after the move.
